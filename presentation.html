Claude Code

AI-агент в терминале. Читает/пишет файлы, запускает команды, работает с git.

Ключевые фичи

  • Полный доступ к файловой системе и терминалу
  • Понимание контекста всего проекта
  • SWE-bench: 80.8% (лидер)
  • 200K контекст (1M в beta)

4 системы расширения

  • Sub-agents -- параллельные специалисты
  • Skills -- lazy-loaded инструкции
  • Hooks -- автоматизация событий
  • MCP -- внешние инструменты

Sub-agents

Специализированные AI-помощники в собственном context window

Researcher

name: research-code model: haiku # Быстрый поиск по коду # Паттерны, зависимости

Implementer

name: implementer model: opus # Реализует по плану # Следует конвенциям

Reviewer

name: reviewer model: sonnet # Баги, уязвимости # CRITICAL / WARNING

Фильтруют output, работают параллельно, не засоряют основной контекст

Skills и Hooks

Skills -- lazy-loaded инструкции

Загружаются только при вызове, не раздувают CLAUDE.md

# ~/.claude/skills/prisma/SKILL.md description: "Prisma migrations" 1. Обнови schema.prisma 2. pnpm prisma migrate dev 3. Проверь SQL 4. Протестируй rollback

Hooks -- автоматизация событий

Bash-команды или LLM-оценка по событиям жизненного цикла

// settings.json { "hooks": { "PostToolUse": [{ "matcher": "Write|Edit", "hooks": [{ "command": "./lint.sh" }] }] } }

Три режима работы

Agent-First

95% задач

  • Знакомые паттерны, CRUD
  • Plan Mode → итерируй
  • Потом execution

Human-First

Нетривиальная логика

  • Маленькие шаги + review
  • Коммит после каждого шага
  • Пропусти Plan Mode

Manual + AI

Новая архитектура

  • Пиши сам
  • LLM для вопросов
  • Понимай каждую строку

Workflow: Debug

Быстрый (1 промпт)

Ошибка: POST /api/orders → 500 Error log: [paste error trace] Последние изменения: git log --oneline -5 Воспроизведение: curl -X POST localhost:3000/api/orders Найди причину и предложи фикс.

Глубокий (multi-step)

1. Сбор информации
"Исследуй, пока не фикси"
2. Анализ root cause
"Ранжируй по impact"
3. Исправление
"Фикс N1 + тест"

Workflow: Code Review

Self-review перед PR

Review git diff main...HEAD Checklist: [ ] Нет hardcoded значений [ ] Error handling для внешних вызовов [ ] Нет секретов в коде [ ] Тесты: happy path + edge cases [ ] Нет breaking changes в API [ ] Типы корректны (нет any) Формат: CRITICAL / WARNING / SUGGESTION

Review чужого PR

Прочитай PR #123: gh pr view 123 --json body,diff Оцени: 1. Решает ли заявленную проблему? 2. Баги или edge cases? 3. Соответствует архитектуре? 4. Достаточно тестов? Напиши review для GitHub.

Золотое правило

git commit после каждого шага /clear после каждой подзадачи

Почему commit

  • Агент "хорошо начинает, плохо заканчивает"
  • Без коммитов откат невозможен
  • Squash потом

Почему /clear

  • Контекст деградирует со временем
  • Агент путает файлы, забывает инструкции
  • Заново прочитать < мутный контекст

Комбинирование инструментов

Claude Code -- основной агент + Sub-agents -- параллельные специалисты ├ researcher (haiku) -- поиск по коду ├ implementer (opus) -- реализация └ reviewer (sonnet) -- code review + MCP серверы -- внешние инструменты ├ context7 -- документация ├ claude-mem -- persistent memory └ github -- issues, PRs + Skills -- lazy инструкции + Hooks -- автоматизация

Hooks не занимают контекст -- они работают детерминистически

Контекст решает всё

Каждый промпт должен содержать три компонента:

WHAT

Что за проект, стек, структура

Next.js 14, TypeScript, Prisma, Monorepo

WHY

Зачем нужно изменение, бизнес-задача

Таблица orders 100k+ записей, нужна пагинация

HOW

Как сделать, ограничения, формат

Cursor-based, по createdAt+id, max 100

Правило: если промпт подходит к тысяче других проектов -- он слишком расплывчатый

CLAUDE.md

"Онбординг" агента в проект. Единственный файл, попадающий в каждую сессию.

Структура

## WHAT (стек и структура) Next.js 14, TypeScript, Prisma Monorepo: apps/web, apps/api ## WHY (контекст проекта) SaaS для управления заказами ## HOW (как работать) pnpm dev / pnpm test Ветки: feature/TICKET-123-desc Коммиты: conventional commits

Progressive Disclosure

CLAUDE.md # 60-100 строк agent_docs/ ├ architecture.md # архитектура ├ testing.md # тесты └ api.md # конвенции API

Max 150-200 инструкций. Остальное -- в agent_docs/ и skills.

Управление контекстом

Context window -- рабочая память агента. Чем больше мусора, тем хуже качество.

  • /clear после каждой подзадачи -- дешевле прочитать заново, чем работать с мутным контекстом
  • MCP -- скрытый потребитель: описания инструментов занимают до 50% контекста ещё до начала работы
  • Sub-agents для фильтрации: вместо 200 строк test output делегируй sub-agent, который вернёт только failures
  • Skills вместо CLAUDE.md: загружаются по требованию, не раздувают постоянный контекст

Few-shot и итеративность

Показывай примеры

Преобразуй API endpoints в интерфейсы: Пример: GET /users/:id → interface GetUserParams { id: string } POST /users body: {name, email} → interface CreateUserBody { name: string; email: string } Теперь для: GET /orders/:id/items?page=N&limit=N

Итерируй, не надейся на one-shot

  • Получил результат
  • Определил, что не так
  • Уточнил конкретно:
    "Используй async/await вместо callbacks"
  • Контекст мутный? /clear и заново

Топ anti-patterns

  • "Сделай всё сам" -- агент деградирует на длинных сессиях, первые файлы ОК, последние -- мусор
  • Без коммитов -- час работы без единого коммита, потом откат невозможен
  • Без /clear -- одна сессия на весь день, контекст деградирует
  • Раздутый CLAUDE.md -- 500 строк инструкций, Claude начинает игнорировать
  • Copy-paste кода -- 200 строк в промпт вместо "прочитай файл X"
  • Все MCP включены -- 10 серверов съедают 50% контекста до начала работы
  • Без Plan Mode -- сразу код, потом переделывать дороже
  • "Агент как Google" -- тратишь контекст на общие знания вместо задач
  • Не проверять output -- LLM галлюцинируют, особенно с деградацией контекста
  • Антропоморфизм -- "Ты гениальный эксперт..." бесполезный fluff

Чеклист для старта

  • Установи Claude Code: npm install -g @anthropic-ai/claude-code
  • Создай CLAUDE.md в проекте: WHAT / WHY / HOW (60-100 строк)
  • Подключи context7 для актуальной документации
  • Подключи claude-mem для persistent memory
  • Освой три команды: /clear, /compact, git commit
  • Начни с Agent-First режима для типовых задач
  • Коммить после каждого шага, /clear после каждой подзадачи
  • Создай первый sub-agent: test-runner на Sonnet
  • Попробуй skills для повторяющихся операций
  • Настрой мониторинг: ccstatusline для status bar, CCM для dashboard
  • Через неделю -- ревью своего CLAUDE.md, убери лишнее
