# LLM Wiki -- инструкции для агента

## Что это за проект
Obsidian wiki по использованию LLM-агентов в разработке и проектной деятельности. Русский язык + английские термины.

## Главный документ
**Перед любыми изменениями прочитай `WIKI-MANIFEST.md`** -- он содержит все правила: структуру, формат страниц, правила ссылок, операции добавления/удаления/обновления.

## Структура
```
README.md            -- главная страница (MOC), точка входа
WIKI-MANIFEST.md    -- конституция: все правила работы с wiki
Инструменты/        -- используемые инструменты (Claude Code, MCP, GSD, ...)
Изучить/            -- интересное, но не освоенное
Практики/           -- гайды, воркфлоу, сравнения моделей
```

## Критичные правила (коротко)
1. Ссылки на источники -- INLINE рядом с текстом, не в конце файла
2. Шапка каждого файла: `# Название` + blockquote + `Назад: [[README]]`
3. Подвал: `Связанные: [[...]]`
4. При добавлении/удалении страниц -- обновить таблицу на `README.md`
5. Без эмодзи, без подпапок, без голых URL
