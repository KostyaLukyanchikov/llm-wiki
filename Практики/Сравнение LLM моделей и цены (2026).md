## 1. Claude (Anthropic) ([pricing](https://platform.claude.com/docs/en/about-claude/pricing) | [benchmarks](https://www.datastudios.org/post/claude-sonnet-4-6-vs-opus-4-6-2026-comparison-capability-split-output-ceilings-long-context-beha))

### Claude Opus 4.6 -- флагман

| Параметр | Значение |
|---|---|
| Context window | 200K (1M в beta) |
| Max output | 128K tokens |
| API input | $5 / 1M tokens |
| API output | $25 / 1M tokens |
| Batch API | 50% скидка |
| Prompt caching | до 90% экономии на input |

**Сильные стороны для кодинга:**
- Лучший результат на SWE-bench Verified: **80.8%**
- Terminal-Bench 2.0: лидер среди всех моделей
- Глубокое понимание архитектуры, умеет работать с большими кодовыми базами
- Отличный agentic coding -- может самостоятельно планировать, редактировать файлы, запускать команды
- Огромное контекстное окно позволяет загрузить весь проект целиком

**Слабые стороны:**
- Самый дорогой из моделей Claude
- Reasoning tokens в extended thinking увеличивают стоимость
- Fast mode стоит 6x от обычной цены

**Когда использовать:** Сложные рефакторинги, архитектурные решения, работа с большими кодовыми базами, задачи где качество критично.

### Claude Sonnet 4.6 -- лучшее соотношение цена/качество

| Параметр | Значение |
|---|---|
| Context window | 200K (1M в beta) |
| Max output | 128K tokens |
| API input | $3 / 1M tokens |
| API output | $15 / 1M tokens |

**Сильные стороны:**
- SWE-bench: **79.6%** -- почти как Opus за 60% стоимости

- Computer use: 72.5% -- практически идентично Opus (72.7%)

**Когда использовать:** Повседневная разработка, code review, написание фич, дебаг. Основная рабочая лошадка.

## 2. OpenAI ([pricing](https://openai.com/api/pricing/))

### GPT-4o

| Параметр | Значение |
|---|---|
| Context window | 128K |
| API input | $2.50 / 1M tokens |
| API output | $10 / 1M tokens |

Хорошая универсальная модель. Для кодинга уступает Claude Sonnet 4.6, но дешевле по input. Хорошо справляется с генерацией кода, но хуже с пониманием больших кодовых баз.

### GPT-4.1

| Параметр | Значение |
|---|---|
| Context window | 1M tokens |
| API input | $2 / 1M tokens |
| API output | $8 / 1M tokens |

Более новая и дешевая замена GPT-4o. Контекст 1M tokens. Хорошая модель для задач, где нужен большой контекст при умеренной цене.

### O3 (reasoning model)

| Параметр | Значение |
|---|---|
| Context window | 200K |
| API input | $10 / 1M tokens |
| API output | $40 / 1M tokens |

**Важно:** O-серия использует "reasoning tokens" -- внутренние токены размышлений, которые тарифицируются как output, но не видны в ответе. Запрос с видимым ответом в 500 токенов может потребить 2000+ токенов. Это делает реальную стоимость значительно выше номинальной.

**Когда использовать:** Сложные алгоритмические задачи, математика, задачи требующие глубокого рассуждения. Не для повседневного кодинга -- слишком дорого.

### O4-mini

| Параметр | Значение |
|---|---|
| Context window | 200K |
| API input | $1.10 / 1M tokens |
| API output | $4.40 / 1M tokens |

Дешевая reasoning модель. Хороша для задач, где нужно рассуждение, но бюджет ограничен. С учетом reasoning tokens реальная стоимость выше.

### GPT-5

| Параметр | Значение |
|---|---|
| API input | $1.25 / 1M tokens |
| API output | $10 / 1M tokens |

Флагман OpenAI. Неплохое качество, но на coding бенчмарках уступает Claude Sonnet 4.6 и Opus 4.6.

## 4. Claude Code -- тарификация ([Max Plan](https://claude.com/pricing/max))

### Подписки Anthropic

| План | Цена | Что дает |
|---|---|---|
| Pro | $20/мес | Базовый доступ к Claude + Claude Code |
| Max 5x | $100/мес | 5x лимитов Pro, Claude Code включен |
| Max 20x | $200/мес | 20x лимитов Pro, Claude Code включен |

### Подписка vs API -- что выгоднее?

**Max 20x за $200/мес** -- это примерно эквивалентно:
- ~8M output tokens Sonnet 4.6 по API ($120) -- но Max дает больше flexibility
- ~40M input tokens Sonnet 4.6 по API ($120)
- При активном использовании Claude Code (несколько часов в день) подписка Max окупается быстро

**API выгоднее когда:**
- Нужно интегрировать в CI/CD пайплайн
- Используешь Batch API (50% скидка) или prompt caching (до 90%)
- Непредсказуемое использование -- платишь только за то, что потребил
- Нужен программный доступ

**Max подписка выгоднее когда:**
- Активная ежедневная работа через Claude Code
- Не хочешь считать токены
- Нужен доступ и к web-интерфейсу, и к Claude Code в терминале
- Работаешь с Opus 4.6 (по API он дорогой)

## 5. AI Coding Tools -- сравнение ([DEV.to](https://dev.to/pockit_tools/cursor-vs-windsurf-vs-claude-code-in-2026-the-honest-comparison-after-using-all-three-3gof))

### Cursor

| | |
|---|---|
| Подход | AI внутри IDE (форк VS Code) |
| Free | 2000 completions/мес |
| Pro | $20/мес |
| Business | $40/user/мес |
| Ultra | $200/мес (20x usage) |

**Плюсы:** Полноценная IDE, привычный интерфейс VS Code, Agent Mode с Composer, inline diff. Самый популярный AI coding tool (1M+ пользователей).
**Минусы:** Привязка к их IDE, подписка не дешевая, зависимость от их routing моделей.

### Claude Code

| | |
|---|---|
| Подход | AI агент в терминале |
| Тариф | Через Max подписку или API |

**Плюсы:** Работает с любым редактором, глубокий контекст проекта, Agent Teams (параллельные агенты с общением), лучшие результаты на SWE-bench, самый большой контекст.
**Минусы:** Терминальный интерфейс не для всех, нет визуального diff в IDE, кривая обучения.

### Windsurf

| | |
|---|---|
| Подход | AI-native IDE, "Cascade" система |
| Free | 25 prompt credits/мес |
| Pro | $15/мес (500 credits) |
| Teams | $30/user/мес |

**Плюсы:** Самый дешевый, Arena Mode для сравнения моделей, хороший Flow-подход.
**Минусы:** Меньше пользователей, экосистема слабее Cursor.

### Рекомендация

- **Cursor** -- если хочешь IDE-experience из коробки и готов платить $20/мес
- **Claude Code** -- если работаешь из терминала, нужен глубокий контекст и максимальное качество кода
- **Windsurf** -- если бюджет ограничен и хочешь попробовать AI coding tool
- **Cursor + Claude Code** -- мощная комбинация: Cursor для быстрых правок, Claude Code для сложных задач

## 6. Матрица решений -- какую модель когда использовать

### По задаче:

| Задача | Рекомендуемая модель | Почему |
|---|---|---|
| Архитектурный рефакторинг | Claude Opus 4.6 | Глубокое понимание, большой контекст |
| Повседневный кодинг | Claude Sonnet 4.6 | Лучшее price/quality |
| Автодополнение в IDE | Claude Haiku 4.5 / GPT-4.1 | Быстро и дешево |
| Сложный алгоритм | O3 или Claude Opus 4.6 | Reasoning capabilities |
| Массовая обработка | DeepSeek V3.2 | Ультра-дешево |
| Генерация тестов | Claude Sonnet 4.6 | Хорошо понимает контекст |
| Code review | Claude Sonnet 4.6 | Находит баги, предлагает улучшения |
| Документация | GPT-4.1 / Sonnet 4.6 | Хороший текст, большой контекст |
| Прототипирование | Sonnet 4.6 + Claude Code | Быстро создает рабочий код |
| Дебаг сложного бага | Opus 4.6 | Глубокий анализ |
| Reasoning на бюджете | O4-mini / DeepSeek R1 | Дешевое рассуждение |

### По бюджету:

| Бюджет | Стратегия |
|---|---|
| Минимальный | DeepSeek V3.2 + Gemini Flash для простого, Claude Haiku для кодинга |
| Средний ($20-100/мес) | Claude Max 5x или Cursor Pro + Sonnet 4.6 API |
| Рабочий ($200/мес) | Claude Max 20x для Claude Code + Cursor для IDE |
| Без ограничений | Opus 4.6 через API для сложного, Sonnet 4.6 для остального |
